Mahjong Game (C++ / Qt Project)

This was my final project for the Object-Oriented Programming course, where we developed a fully functional Mahjong Solitaire game. The project combines a C++ backend with a QML-based frontend using the Qt framework. The user can select and match open tile pairs while the game ensures rules are enforced (e.g., tiles that are not open cannot be selected). The game board uses a “turtle layout” similar to traditional Mahjong Solitaire.

Key Features:
- Functional 2D graphical interface built with QML
- Logic and model-view separation implemented in modern C++
- Tile interactions handled through QML bindings to C++ objects
- Dynamic layout generation (Turtle Layout)
- Bonus: Board shuffle option to assist when no moves remain

What I Did:
- Designed the architecture using Qt’s object model, exposing C++ classes (Board, Tile, TileModel) to QML
- Managed game state, tile selection logic, and match validation entirely in C++
- Utilized QML to create an intuitive user interface with responsive tile interactions
- Implemented layout generation and optional reshuffle logic on the backend
- Integrated assets using Qt Resource System for portability

What I Learned:
- How to build desktop applications by combining declarative UI (QML) with imperative logic (C++)
- How to use model-view separation and context properties in Qt
- Advanced object-oriented programming practices, including encapsulation, abstraction, and modular design
- The fundamentals of memory management, meta-object programming, and QML-C++ communication

Optimization of CPFEM Parameters using ML

This research project focused on using machine learning (ML) to optimize parameter calibration in Crystal Plasticity Finite Element Models (CPFEM) for advanced steels. The study aimed to automate the traditionally manual and time-consuming process of calibrating material parameters from nanoindentation data using an ANN model trained on simulation results. My primary contribution was leading the development of the machine learning pipeline and all data engineering steps required for precise predictions.

Key Features:
- Worked with dual-phase steels (DP1000 and QP1200) using nanoindentation data
- Designed data cleaning and preprocessing pipeline (IQR filtering, normalization)
- Developed feature engineering steps for optimal model training
- Trained and evaluated multiple ML models: ANN, CNN, LSTM, and Random Forest
- Achieved high-accuracy calibration with low RMSE and high R² scores (up to 0.99)

What I Did:
- Designed and coded the complete ML pipeline in Python, including data preprocessing, training/testing, and evaluation
- Created custom loss functions and model evaluation routines
- Developed a robust ANN-based system that predicted critical CPFEM parameters (shear stress, hardening rates, etc.) from nanoindentation FD curves
- Integrated percentile-based outputs (25th, 50th, 75th) into model training for improved curve-fitting fidelity

What I Learned:
- Gained deep hands-on experience in real-world application of machine learning to materials engineering
- Learned to handle scientific datasets, reduce noise, and optimize model performance
- Improved teamwork and interdisciplinary collaboration across engineering, simulation, and data science domains
- Developed a greater understanding of predictive modeling, CPFEM workflows, and materials informatics

Beijing Air Quality Prediction (ML Course Project)

This machine learning course project involved building a predictive model for Beijing's Air Quality Index (AQI) using historical environmental and pollutant data. The goal was to create a regression model that could accurately forecast daily AQI values based on variables like PM2.5, temperature, humidity, and wind speed.

Key Features:
- Dataset sourced from UCI (Beijing PM2.5, 2010–2014), processed into daily aggregates
- Used two regression models: Random Forest Regressor and Ridge Regression
- Preprocessing included handling missing data and Min-Max normalization
- Evaluated models using Mean Squared Error (MSE) on train, validation, and test sets
- Performed visual data analysis, including feature distributions and correlation matrix

What I Did:
- Cleaned and preprocessed over 8,000 daily observations from the original hourly dataset
- Trained and evaluated both ML models using a 75/10/15 data split
- Applied Ridge Regression to mitigate overfitting and achieve lower MSE
- Analyzed model performance using MSE metrics and selected Ridge Regression as the final model
- Visualized correlations and feature distributions using Python (Seaborn, Matplotlib)

What I Learned:
- Gained practical experience applying regression models to real-world environmental data
- Learned the importance of proper feature scaling and model validation - Developed the ability to compare different model behaviors and interpret performance metrics - Practiced thoughtful feature selection and preprocessing to improve model generalization
